# Release Notes — 2026-05-13 → 2026-05-17

12 commits since `a6eb8b9`. Highlights below; commit hashes are included for reference.

## New: Audience Analysis

A third analysis surface added to `/admin/analysis` that profiles a production's paid attendees against a chosen comparison group of theaters and against the whole facility.

- **Per-window metrics** for 3mo / 6mo / 1yr / 3yr / 5yr / Ever: first-time, returning, *dedicated* (attended every overlapping production in the comparison group), 2+ in comparison group, and 3+ in building. Visit semantics use production-run overlap, not order date, so a show whose run straddled a window boundary is credited correctly. (`05ca1bf9`)
- **Lifetime returning-attendee headline rows** highlight the three most recent prior productions in the comparison group plus a catch-all "any production" row.
- **Per-cohort CSV export.** Every numeric count in the results table is now a clickable TRG Arts export. Click queues an `AudienceCohortExport` Resque job that produces a FileStore CSV emailed to the user and listed on the reports page. (`3b264808`)
  - TRG CSV schema: `:Segment` carries the three-letter list-type code (`LST`), `:Title` is the segment label (capped at 50 chars), `:ClientPatronID` carries the per-theater external_id, `:StagemgrPatronID` always carries `address.id`.
  - Email column gated by `can?(:view_email, Address)` **OR** the address being in any relevant MyEmma attendee group (`0b3fca4b`). Mirrors the Production Attendee report rule and adds an `OptedInForEmail` Y/N column.
  - Facility-scope cohorts ("vs. facility" rows) are admin-only.
  - Cells whose count equals the full cohort size are suppressed — exporting them would just reproduce the cohort itself.
- **Back-link round-trips comparison theaters** via `comparison_theater_ids` so operators can iterate without re-picking theaters. New "Audience Analysis" button on the admin production show page jumps to the form preselected and seeded. (`67ab98dc`)

## New: Theater tags

Free-form tags on theaters with autocomplete and pill UI. (`6fde92c8`)

- Tagify input on the theater edit form: pills with X delete controls, autocomplete from existing tags, Enter or comma to commit.
- Tags render on the admin theaters index (next to the name) and on the theater detail page.
- Theaters index search box now matches against tag names.

Integrates with the audience analysis flow: the rate-of-sales autocomplete now offers **"All shows tagged X"** group entries derived from theater tags, deduplicated case-insensitively, still filtered by the user's CanCan production scope. (`9db3cd9b`, docs in `88a4769c`)

## Fixes

- **jQuery UI styling restored.** The SCSS `@import` of jquery-ui-rails' `jquery-ui.css` resolved to a Sprockets manifest comment which Sass inlined as inert text. The compiled `application.css` contained zero `.ui-datepicker` rules, so every date picker rendered unstyled. Now loaded as a top-level Sprockets asset; legacy 2012-era cupertino theme trees and three dead duplicate rule blocks pruned. (`689c9856`)
- **Donation list export includes donor emails.** Report relied on `MailingList.mailing_hash_from_buyer`'s default `allow_email_export=false`, so every donor row's Email column was blank — defeating the fundraising follow-up workflow this export exists for. (`fd204520`)

## Infrastructure / dev environment

- **my_emma 0.1.3 → 0.2.1** with new `read_only!` mode. Development without Emma credentials now switches the gem to read-only instead of fully disabling it, so dev can hit real Emma reads without risk of writes. Transitive: httparty 0.7.8 → 0.24.2, new multi_xml 0.9.1. (`7c13187a`)
- **Logger NameError on Rails CLI commands fixed.** httparty 0.24.2 no longer eager-loads stdlib `Logger`, so `rails c`, `credentials:edit`, `rails s`, and Resque workers crashed with `NameError: uninitialized constant ActiveSupport::LoggerThreadSafeLevel::Logger`. Added explicit `require 'logger'` in `config/boot.rb`. (`70c0d1f0`)
- **Cucumber stability.** Firefox 150 on macOS does not always exit when WebDriver `quit()` is called; a stale browser holds the Marionette port and poisons every subsequent `@javascript` scenario with `InvalidSessionIdError`. `AfterConfiguration` hook now `pkill`s lingering Firefox/geckodriver before the first scenario. Restored 46 cascade-failing scenarios. (`1a7bd584`)
